# OpenFFBoard 编译与打包指南 (macOS & Windows)

本指南旨在指导开发者在 macOS 和 Windows 环境下完成 OpenFFBoard 固件的编译以及 Configurator 上位机的打包。

---

## 1. 固件编译 (Firmware Compilation)

OpenFFBoard 固件基于 STM32F407，使用 `GNU Arm Embedded Toolchain` 进行编译。

### 🍎 在 macOS 下编译 (推荐)
macOS 是类 Unix 系统，非常适合嵌入式开发。

1.  **安装工具链**：
    使用 Homebrew 安装 GCC 交叉编译器：
    ```bash
    brew tap ArmMbed/homebrew-formulae
    brew install arm-none-eabi-gcc
    brew install make
    ```
2.  **获取源码**：
    ```bash
    git clone --recursive https://github.com/OuIChien/OpenFFBoard.git
    cd OpenFFBoard/Firmware
    ```
3.  **编译**：
    直接运行 `make`：
    ```bash
    make -j$(sysctl -n hw.ncpu)
    ```
    编译完成后，在 `build/` 目录下会生成 `FFBoard.bin` 和 `FFBoard.dfu`。

### 🪟 在 Windows 下编译
1.  **安装工具链**：
    - 下载并安装 [GNU Arm Embedded Toolchain](https://developer.arm.com/downloads/-/gnu-arm-embedded-toolchain-downloads)。
    - 安装 [Make for Windows](https://gnuwin32.sourceforge.net/packages/make.htm) 或使用 Git Bash 自带的工具。
2.  **编译**：
    在 `Firmware` 目录下打开命令行（如 PowerShell 或 CMD）：
    ```cmd
    make -j4
    ```

## 2. 上位机打包 (Configurator Packaging)

Configurator 是基于 Python 和 PyQt5 开发的。

### 🍎 在 macOS 下运行与打包
1.  **环境准备**：
    ```bash
    cd Configurator
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```
2.  **打包成 .app**：
    使用 `PyInstaller` 或 `Nuitka`：
    ```bash
    pip install pyinstaller
    pyinstaller --name "OpenFFBoard-Config" --windowed main.py
    ```

### 🪟 在 Windows 下打包 (发布正式版常用)
1.  **使用提供的脚本**：
    项目中已经包含了 `build.bat`。确保已安装 Python 环境：
    ```cmd
    cd Configurator
    .\install_requirements.bat
    .\build\build.bat
    ```
    这会生成一个 `.exe` 文件。

## 3. 烧录固件 (Flashing)

### 通用方案：DFU 模式
1.  将 OpenFFBoard 拨至 DFU 模式（通常是跳线或按住 Boot0 键）。
2.  使用 **STM32CubeProgrammer**（支持 Mac/Win）。
3.  选择 USB 模式，连接后加载 `FFBoard.bin` 烧录。

### macOS 命令行方案：
安装 `dfu-util`：
```bash
brew install dfu-util
dfu-util -a 0 -s 0x08000000:leave -D build/FFBoard.bin
```

## 5. 推荐 IDE 与开发环境 (IDE Recommendations)

针对 OpenFFBoard 这种包含 **C++ 固件**与 **Python 上位机**的混合项目，推荐以下开发环境：

### 🚀 首选：Visual Studio Code (VS Code) - 跨平台/轻量
这是目前最推荐的方案，适合 macOS 和 Windows。
1.  **必装扩展**：
    - `C/C++` (Microsoft): 提供代码补全与跳转。
    - `Python` (Microsoft): 用于 Configurator 开发。
    - `Makefile Tools` (Microsoft): 自动识别 OpenFFBoard 的编译结构。
2.  **配置技巧**：
    - 在 VS Code 中打开 `Firmware` 文件夹。
    - 在左侧 `Makefile` 面板中，将 **Makefile Path** 指向 `./Makefile`。
    - 这将使 IDE 能够正确索引 `arm-none-eabi` 工具链的头文件，消除代码报错。

### 🛠 专业：CLion (JetBrains) - 深度分析
如果你需要极强的代码重构和静态检查（防止马达控制逻辑中的内存风险）：
- CLion 能够完美处理 Makefile 项目。
- 支持直接连接 ST-LINK 进行图形化硬件调试。

### ⚡️ 官方：STM32CubeIDE - 硬件调试
- 基于 Eclipse，对 STM32 的寄存器查看和单步调试支持最稳。
- 源码根目录已包含 `.project` 文件，可直接导入。
