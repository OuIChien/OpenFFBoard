# RobStride RS04 协议使用与测试指南

本指南详细说明了在 OpenFFBoard 中安装并烧录固件后，如何配置并使用 **MIT 协议** 和 **私有通信协议**。

---

## 1. 初始连接与检查

在烧录完包含 RS04 驱动的固件后：
1.  **物理连接**：确保控制板的 CAN-H/L 与 RS04 的 CAN 接口正确对接，且总线阻抗匹配（120Ω）。
2.  **启动上位机**：打开 `OpenFFBoard Configurator`。
3.  **选择驱动**：在 **Motor Driver** 选项中，从下拉菜单选择 `RobStride RS04 (Ax1)` 或 `(Ax2)`。
4.  **校验连接**：在控制台输入以下命令确认电机是否在线：
    ```bash
    rs04.connected
    ```
    返回 `1` 表示通信正常。

---

## 2. 协议切换方法

你可以通过命令实时切换协议（切换后建议重新点击 **Save to Flash** 并重启马达以同步状态机）。

### 🛠 模式 A：MIT 协议 (推荐用于力回馈)
此模式下，控制权完全交给 OpenFFBoard，能够实现极高的动态响应。
- **设置命令**：
  ```bash
  rs04.protocol 0
  ```
- **特点**：
  - 支持 **P-V-T 混合控制**。
  - 能够直接利用电机的 **KD 参数**（微分增益）来实现物理级的阻尼效果（Damping），手感极佳。
  - 数据包极小，延迟低。

### 🛠 模式 B：私有通信协议
此模式使用 RS04 的默认扩展帧协议，适合参数调试或传统的电流/力矩控制。
- **设置命令**：
  ```bash
  rs04.protocol 1
  ```
- **特点**：
  - 兼容官方 **MotorStudio** 的通讯逻辑。
  - 适合用于检查电机内部的详细寄存器状态或进行固件更新。

## 3. 飞行模拟手感初探

作为工业工程师，我建议你按以下步骤进行初次手感测试：

### 第一步：设置安全力矩
先将最大力矩限制在一个安全范围，防止操纵杆意外剧烈震动：
```bash
rs04.maxtorque 300
```
*(表示限制在 3.0 Nm)*

### 第二步：测试“空载”阻力
1.  切换至 **MIT 协议**。
2.  在飞行模拟器未运行时，缓慢推拉操纵杆。
3.  如果感觉到步进感或明显的摩擦力，可以通过调整 `KD` 参数来补偿。

### 第三步：模拟气动回正 (Centering Force)
1.  启动 **X-Plane 12** 或 **MSFS 2020**。
2.  在跑道上静止时，你应该能感觉到操纵杆比较轻，随着起飞速度增加，操纵杆的“重感”应随之增强（这是由 OpenFFBoard 的 FFB 算法通过 RS04 实现的）。

---

## 4. 故障排查

- **命令无响应**：检查 `rs04.canid` 是否与电机内部设置的 ID 一致（RS04 默认为 1）。
- **电机震动（飞车）**：这通常是 PID 参数过高或编码器反向导致。请立即断电，并检查 OpenFFBoard 的 **Invert Axis** 选项。
- **协议无法切换**：请确认电机是否处于“使能”状态。部分固件版本要求在电机未使能（Disarmed）时才能切换底层协议。
