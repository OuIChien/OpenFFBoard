# RobStride RS04 私有协议开发与集成指南

本指南旨在详细说明如何在 OpenFFBoard 生态系统中集成 RobStride RS04 伺服电机的私有协议（Private Protocol），涵盖固件（C++）与配置器（Python/PyQt6）的协同工作原理。

---

## 1. 固件架构实现 (`RobStrideRS04.h/cpp`)

RS04 的集成采用了多重继承模式，使其能够无缝接入 OpenFFBoard 的各个子系统：

*   **`MotorDriver`**: 提供 `turn()`、`startMotor()`、`stopMotor()` 接口，对接 FFB 算法输出的力矩需求。
*   **`Encoder`**: 将电机内置编码器反馈的数据（位置、速度）上报给系统，作为力回馈闭环的基础。
*   **`CanHandler`**: 实现底层 CAN 消息的收发与过滤逻辑。
*   **`CommandHandler`**: 定义并处理来自 PC 端的串行指令（如修改 CAN ID、切换协议等）。
*   **`PersistentStorage`**: 负责将电机配置（CAN ID、最大扭矩、协议选择）保存至 STM32 的 Flash 中。
*   **`cpp_freertos::Thread`**: 独立线程运行，执行心跳包发送、掉线检测及初始化发现。

### 1.1 协议双模切换逻辑
RS04 支持 **MIT 模式** 和 **Private（私有）模式**。
*   **MIT 模式**: 使用标准 11 位 CAN ID。结构紧凑，但通信功能受限。
*   **Private 模式**: 使用扩展 29 位 CAN ID。支持更丰富的数据反馈和指令，采用类似小米 CyberGear 的通信格式：
    *   `Type (5bit)` | `Master ID (8bit)` | `Motor ID (8bit)`

### 1.2 关键通信函数
*   `sendTorquePrivate()`: 将 16 位力矩值映射并封装进扩展帧。
*   `canRxPendCallback()`: 这是一个高性能回调，自动根据帧类型（标准或扩展）切换解码算法，实时更新 `lastPos`、`lastVelocity` 和 `lastTorqueFeedback`。
*   `Run()`: 线程主循环。当处于 Private 模式且未检测到连接时，会持续发送 `Type 0` 握手包进行设备发现。

## 2. PC 端配置器实现 (`rs04_ui.py`)

配置器界面作为用户与硬件交互的桥梁，通过 `CommunicationHandler` 与固件进行二进制/文本命令交换。

*   **双向同步机制**: 
    *   **SET**: UI 控件（如 `spinBox`）发生变化时，通过 `send_value("rs04", "canid", val)` 发送指令。
    *   **GET/CALLBACK**: 固件反馈数据后，通过 `register_callback` 注册的函数（如 `canIdChangedCallback`）更新 UI，确保状态一致。
*   **实时监控**:
    *   UI 包含一个 `QTimer`，每秒循环查询 `connected` 状态。
    *   `updateConnectedStatus` 函数根据反馈动态调整界面的颜色和文字提示。

## 3. 指令集定义 (Command Set)

配置器与固件之间通过以下指令进行通信：

| 命令 | 描述 | 数据类型 | 备注 |
| :--- | :--- | :--- | :--- |
| `rs04.canid` | 设置/读取电机 CAN ID | `uint8` | 修改后固件会自动重设 CAN 过滤器 |
| `rs04.protocol` | 切换协议模式 | `enum` | 0: MIT, 1: Private (需要重启生效) |
| `rs04.maxtorque` | 设定最大安全扭矩 | `float` | UI 传输时会放大 100 倍转为整数 |
| `rs04.connected` | 读取电机连接状态 | `bool` | 固件 500ms 未收到回包则判定为断开 |

## 4. 开发与调试建议

1.  **CAN 速率匹配**: RS04 建议运行在 **1Mbps**。固件中的 `Run()` 函数会强制尝试设置 CAN 预设速率。
2.  **力矩单位映射**: 
    *   FFB 算法输出的 `power` 范围是 `-32767` 到 `32767`。
    *   映射公式：`torque = (power / 32767.0) * maxTorque`。
    *   RS04 的协议最大物理量程通常为 ±120Nm，但在映射时应根据 `maxTorque` 参数限制在安全范围内。
3.  **编码器解析**: RS04 返回的 `pos` 为弧度制。在 `getPos()` 中，程序将其转换为 OpenFFBoard 通用的 16 位整数计数，转换因子为 `10430.378f` (即 $65536 / 2\pi$)。

## 5. 扩展性说明

若需增加新的反馈参数（如电流、总线电压）：
1.  在 `RS04Commands` 枚举中增加新项。
2.  在 `registerCommands()` 中注册该项。
3.  在 `canRxPendCallback()` 中解析 CAN 数据包对应位。
4.  在 `command()` 函数中添加 `case` 分支处理查询。
5.  在 `rs04_ui.py` 中添加对应的控件和回调处理。
